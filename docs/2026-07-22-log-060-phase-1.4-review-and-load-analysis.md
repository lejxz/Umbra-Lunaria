## Work Completed

### Bugs found and fixed

1. **CWL side normalization** (`lib/ingest/war-sync.ts` `syncCwlWars`):
   The `/clanwarleagues/wars/{warTag}` response lists the two clans as `clan`
   and `opponent`, but does NOT guarantee our clan is `clan`. When our clan
   was the `opponent`, `syncCurrentWar` stored the wrong side as "ours" —
   wrong `ownStars`, wrong `war_participants` (FK to `members`), and a swapped
   snapshot. **Fix:** detect which side is ours and swap `clan`/`opponent`
   before calling `syncCurrentWar`. Also return the actual CWL `lastState`
   (preparation/inWar/warEnded) so the refresh response doesn't hardcode
   `"inWar"`.

2. **Vanishing-war bug** (`lib/db/war-queries.ts` `getWarCenter`):
   When there was no active war, the old code picked the most-recent ended war
   as the hero "current war". If that row had no snapshot (a war-log backfill
   row), it couldn't render the roster/attack detail AND was excluded from the
   history list — so it vanished from the UI entirely. **Fix:** only an active
   war (preparation/inWar) becomes the hero; ended wars always appear in
   history. Added a `lastResult` field (most-recent ended war) surfaced as a
   one-line "Last result" card in the hero's no-active-war state so the page
   still answers "what was the last war?".

3. **History query over-fetch** (`lib/db/war-queries.ts`):
   `getWarCenter` selected `*` (including the full `war_snapshot` JSONB) for
   50 history rows. A live-tracked war snapshot can be tens of KB. **Fix:**
   project only the columns the history list needs + a `hasSnapshot` boolean
   (`war_snapshot IS NOT NULL`). Cuts the query payload from ~50× JSONB to 50
   scalars.

4. **Dead code + stale import** (`lib/ingest/war-sync.ts`):
   `getLatestWarRow` was unused (the War Center reads via `getWarCenter`).
   Removed it and the now-unused `desc` import.

### Load analysis

New doc: `docs/2026-07-22-neon-vercel-free-tier-load-analysis.md`. Uses live
DB measurements (9.4 MB total, ~495 snapshots over 2 days, 51 wars) to project
Neon storage/compute and Vercel function-invocation/duration consumption over
30/90/365 days. Key finding: the **first ceiling to hit is Vercel Hobby's
10-second function duration on the daily batch** (~8s at 5 members, crosses
10s at ~8 members). Neon storage (~110 MB projected at 1 year) and Vercel
invocations (~5k/month vs 100k limit) have ample headroom. Recommendations
include snapshot pruning for departed members and setting `maxDuration = 60`
on the ingest route before the clan grows past ~8 members.

## Decisions Made
- **Only active wars are the hero.** Ended wars live in history only. This
  fixes the vanishing-war bug and keeps the hero honest about "is there a war
  right now?". The `lastResult` card in the no-active-war state preserves the
  "what was the last war?" answer.
- **CWL side normalization happens before `syncCurrentWar`.** Centralizing the
  swap in `syncCwlWars` means `syncCurrentWar` and the snapshot parser don't
  need to know which side is "ours" — they always see our clan as `clan`.
- **History query projects columns + a boolean.** Pulling `war_snapshot IS
  NOT NULL` instead of the JSONB itself is a standard optimization; the
  `hasDetail` badge on history rows only needs the boolean.
