# Hydration Errors & War Card UI Fixes (2026-07-22)

## The Issue
The application was throwing React Error #418 (Hydration mismatch) in production and development environments. This occurs when the initial HTML generated by the Next.js server does not exactly match the DOM structure rendered by the React client on the first pass.

Additionally, the `CurrentWarCard` UI needed some visual polish: the `5v5` badge looked too plain, and the war stats (stars/destruction) were taking up too much vertical space.

## Changes Made

### 1. Fixed Hydration Errors in `TimeAgo` Component
The `<TimeAgo>` component was originally written to render `Date.now()` differences and localized strings (`toLocaleString`) directly during the first render. 
- **Server:** Runs in its own timezone and calculates `Date.now()` at render time.
- **Client:** Runs in the user's browser timezone and calculates `Date.now()` milliseconds or seconds later.
This mismatch caused Hydration Error #418.
- **Fix:** We introduced a `mounted` state using `useState` and `useEffect`. On the initial render, the component now outputs a generic time or neutral string (which matches the server), and only formats the localized, relative time once the component has successfully mounted on the client. We also added `suppressHydrationWarning` to the `<time>` tag to safely suppress expected text mismatches on the client.

### 2. Fixed Hydration Errors in `DashboardShell` (PollTime & Chip components)
Similar to `TimeAgo`, the `dashboard-shell.tsx` component was formatting dates using `.toLocaleString()` and calculating a live countdown during the initial render.
- **Fix:** Added a `mounted` state check before displaying the localized dates and the active countdown. We also strategically applied `suppressHydrationWarning` to the `<span>` elements wrapping the dynamic values to prevent any flashing or strict hydration warnings.

### 3. CurrentWarCard Polish
- **Stats Layout:** The Stars and Destruction Percentage were refactored into a single baseline row (e.g., `★24` `50%`) with the percentage rendered as smaller subscript-like text to the right. This saves vertical space and looks more like standard Clash of Clans war interfaces.
- **5v5 Text Design:** Reverted the plain `5v5` text back into a sleek, purple glowing pill badge (`V` is capitalized and generously letter-spaced) that sits elegantly above the crossed swords icon.

## Important Note on Hydration
When building React components in Next.js, **never** render `Date.now()`, `Math.random()`, `window`, or browser-specific timezone methods (like `toLocaleString`) directly in the component's main render body without a `mounted` check or `suppressHydrationWarning`.
