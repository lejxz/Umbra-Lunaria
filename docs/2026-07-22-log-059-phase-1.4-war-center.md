## Decisions Made
- **Snapshot column over joining opponent members.** Opponent members cannot live in `war_participants` (FK to `members`), so the full `CocCurrentWar` is stored as JSONB on `wars`. The roster + attack log are parsed from the snapshot for the UI; `war_participants`/`war_attacks` still serve own-clan scoring and member-detail war history. Backfilled war-log rows have a null snapshot (result/destruction only) and are marked `detailed: false` in history.
- **45s refresh TTL (within the concept's 30–60s band).** Module-level cache collapses concurrent refresh bursts; the response reports `cached: true` + remaining seconds so the UI shows a live "try again in Ns" countdown rather than silently no-op'ing.
- **TH mismatch cue = ≥2 levels, prep only.** Flagged per map position as a discussion aid; never used to assign attacks (concept/07 explicitly forbids autonomous assignment).
- **CWL ingestion is best-effort and only attempted when regular currentwar is `notInWar`.** Avoids an extra league-group fetch every 10-minute poll when the clan is in a normal war. 404 (not in CWL season) is swallowed, not surfaced as an error.
- **No fixture/DB-integration tests added.** Per the project's standing test-strategy note (Step 1.0.A), DB-integration tests are deferred; Phase 1.4 was verified live via Agent Browser against the active preparation war and by direct script calls (`refreshCurrentWar`, `backfillWarLog`) against the live Neon DB.
