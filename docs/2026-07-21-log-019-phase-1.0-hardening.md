## Decisions Made

- **ESLint 9 + FlatCompat over `@eslint/eslint-plugin`**: `eslint-config-next` ships in legacy eslintrc format. Used `@eslint/eslintrc`'s `FlatCompat` to bridge it into the flat config array rather than waiting for a native flat-config release.
- **Non-destructive migration strategy**: Instead of `drizzle-kit push` (which required interactive conflict resolution for renamed columns), wrote a hand-crafted additive SQL migration that adds columns/tables/indexes without dropping data. The old `badge_url` (text) and `location` (text) columns were renamed to `_old` suffixes rather than dropped, preserving backward compatibility.
- **War idempotency via unique index**: Chose a database-level unique constraint on `(war_id, attacker_tag, attack_order)` over application-level dedup. The `.onConflictDoNothing()` approach is simpler and race-safe.
- **Opponent members not persisted to `war_participants`**: The table has a FK to `members.player_tag` and opponent tags aren't in our `members` table. Defender identity is preserved via `war_attacks.defender_tag` and opponent aggregate stats live on the `wars` row. This is a deliberate schema-correct choice, documented in a code comment.
- **Parallel subagent execution**: Tasks 1.0.B and 1.0.D were delegated to parallel subagents since they touch disjoint file sets (`components/ui/*` vs `app/api/*` + `lib/`). The only conflict was `lib/scoring/donations.ts` (created by 1.0.D, lightly patched by 1.0.B for `noUncheckedIndexedAccess` guards) — resolved cleanly.
- **`neon()` array queries**: Discovered that `sql\`ANY(${array})\`` doesn't work with the neon serverless driver's parameterized queries. Later (in 1.1) switched to drizzle's `inArray()` helper.
