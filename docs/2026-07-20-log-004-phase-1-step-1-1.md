# Log 004 — Phase 1 Step 1.1 Main Dashboard

**Date:** 2026-07-20
**Time:** 05:35 PM (+08:00)

## Summary of Session

Implemented Phase 1, Step 1.1 as a data-backed dashboard using the approved Umbra Lunaria design proposal. The dashboard now reads clan, member, snapshot, and war data through reusable queries and presents the summary, contribution, activity, attention, roster-history, profile, and navigation sections.

## Work Completed
- Replaced the dashboard placeholder with a responsive glassmorphic dashboard matching the proposal’s typography, colors, stat cards, and panel layout.
- Added donation windows for 24 hours, 7 days, and 30 days with contribution totals and a top-donor list.
- Added an activity timeline with an explicit cold-start empty state.
- Added clan profile data, all-time war record/win-rate summary, member count, capital hall level, needs-attention opt-out list, and recent join/leave log.
- Added current-war and capital navigation strips.
- Expanded `lib/db/queries.ts` with dashboard-specific donation, activity, clan-log, and war-participation reads.
- Marked Step 1.1 complete in `concept/12-roadmap-and-modularity.md`.

## Decisions Made
- Used server-side dashboard data loading with `force-dynamic` because clan and snapshot data change as ingestion runs; no read API routes were introduced.
- Computed donation deltas from the earliest and latest snapshots inside each selected window, keeping weekly-reset handling in ingestion as specified by the concept docs.
- Used the existing shared `Tabs`, `StatCard`, `Badge`, and `EmptyState` primitives rather than adding a dashboard-specific component library.
- Kept capital raid details intentionally summarized as a navigation strip because the full raid-weekend endpoint is outside Step 1.1 and remains planned for a later phase.

## Deviations and Verification
- The activity visualization is a lightweight CSS bar timeline rather than Recharts; this keeps the first dashboard pass dependency-light while preserving the required time-window signal and mobile overflow behavior.
- The current war strip reports recorded state and stars but does not calculate a live countdown; that belongs to the dedicated War page and refresh flow.
- `npm run typecheck` passed.
- Database migrations and `npm run build` passed with `.env.local` loaded into the process.
- `git diff --check` passed. No secrets or `.env.local` were staged.
