## Decisions Made

- **Client-side sorting/filtering**: The clan has 5-50 members — small enough to sort/filter client-side with `useMemo`. URL params for sort/filter state were considered but add complexity without benefit for this scale. The concept's "server-safe sorting" requirement is met by the server fetching all data; the client just re-orders it.
- **Member details fetched server-side in parallel**: The page fetches `getMemberDetail()` for all members via `Promise.all`. This is fine for a small clan. For larger clans, a client-side fetch-on-click would be better.
- **Progression cards with unit-icon-map**: Used `getUnitIcon()` from `lib/assets/unit-icon-map.ts` with a text fallback (first letter) for unmapped units. Maxed units get a green border treatment.
- **Rushed analysis shows "Not available yet"**: The cap reference data files (`lib/reference-data/*-caps.json`) are stubs. The rushed section shows the Phase 3.0 placeholder per concept/06.
- **Activity grid as colored squares**: The 30-day activity is shown as a grid of small colored squares (active=purple, inactive=dim) rather than a chart — simpler and more scannable for a per-member view.
